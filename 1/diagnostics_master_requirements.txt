沅興木業－最終母版需求總表（完整整合版）

版本用途：
這份是後續所有修復、診斷、打包、部署前檢查的唯一母版依據。
之後修復時必須以本文件逐項對照，不准只修主要問題，不准漏小問題，不准修一個壞另一個。

============================================================
0. 最高修復原則（硬規則）
============================================================

0.1 修復原則

1. 不要 overlay。
2. 不要 hardlock。
3. 不要用補丁檔硬塞功能。
4. 不要新增 setInterval。
5. 不要新增 MutationObserver。
6. 不要同時保留多個 renderer。
7. 不要同一個按鈕重複綁多個 click。
8. 不要只改 JS，不改 HTML / CSS / app.py / db.py。
9. 所有修改都要直接寫進實際載入主檔：
   - HTML
   - JS
   - CSS
   - app.py
   - db.py
   - requirements.txt
   - wsgi.py
   - render.yaml
   - Procfile
   - PostgreSQL migration
10. 不准改掉原本正常功能。
11. 使用者說圖幾，就只改圖幾，不准亂刪其他頁面按鈕。
12. 診斷書提供後，critical / error / warn / 小問題都要逐項修。
13. 沒修到 100%，必須明確說還有下一包。
14. 不准讓舊版覆蓋新版造成頁面亂跳。
15. 不准修一個壞另一個。
16. 不准把還沒驗證的功能說成已完成。
17. 不准刪掉快取架構：
   - yx_cache.js
   - yx_core.js
   - STATIC_VERSION
   - API timeout
   - 背景保存 queue
   - fast cache
   - service worker 不快取 API
18. 不准把正常功能「收斂掉」。
19. 不准修完後又說要重新收斂。
20. 不准讓新版又被舊版 renderer 蓋回去。
21. 不准因為修診斷頁或修單頁功能而更動沒壞的頁面。
22. 不准用舊版歷史錯誤污染新版診斷結果。
23. 診斷如果還有 current-version critical / error / warn，不能說 100% 完成。

1. 全系統共同資料規則
============================================================

1.1 前端先顯示，後端背景保存

所有操作都要：

1. 前端立刻顯示。
2. 寫入本機資料狀態。
3. 放入背景保存 queue。
4. 後台寫入 DB。
5. 成功後對齊資料庫。
6. 失敗要提示並保留重送。
7. 切頁不能中斷。
8. 重新整理不能消失。
9. 關掉頁面後，下次開啟要能繼續保存。
10. 不可只顯示成功但 DB 沒寫入。
11. 不可因 API 回空而把本機已有資料洗掉。
12. 不可因 API timeout 而清空畫面。

9. 材積公式（正式版）
============================================================

9.1 商品格式

長 x 寬 x 高 = 支數拆分

例如：

130x12x063=113x4+112+100

9.2 支數拆分

113x4+112+100

表示：

113x4 = 113 支 × 4 件 = 452 支
112 = 112 支 × 1 件
100 = 100 支 × 1 件

9.3 總支數

452 + 112 + 100 = 664 支

9.4 總件數

4 + 1 + 1 = 6 件

9.5 尺寸換算規則

長：

130 → 1.3
140 → 1.4
80 → 0.8

寬：

12 → 1.2
30 → 3
25 → 2.5

高 / 厚：

063 → 0.63
083 → 0.83
05 → 0.5
12 → 1.2
125 → 1.25

9.6 材積公式

材積 = 總支數 × 長度換算 × 寬度換算 × 高度換算

9.7 範例

130x12x063=113x4+112+100

拆解：

總支數 = 664
長 = 1.3
寬 = 1.2
高 = 0.63

計算：

664 × 1.3 × 1.2 × 0.63
= 652.5792

顯示：

652.58 才

9.8 材積顯示規則

1. 材積不能用件數算。
2. 材積要用總支數算。
3. 件數只用來顯示幾件。
4. 出貨預覽必須顯示公式拆解。
5. 診斷和測試清單要檢查材積是否顯示。

1. 清空 warehouse_cells

5. 清掉格內商品

10.3 格子顯示格式（最新正式版）

格子要長這樣：

1           6+2             8件
永和 DF 132X11X013  6
保固 SP 131X10X013  2

10.4 格子顯示排版規則

1. 第一排要跟第二、三排一樣寬。
2. 第一排不能太寬。
3. 第一排不能用 space-between 拉到過寬。
4. 第一排固定三段：
   - 左：格號
   - 中：件數拆分
   - 右：總件數
5. 第二行開始固定格式：
   客戶名 材質 尺寸 該尺寸件數
6. 客戶名顏色保持現在紅色。
7. 材質顏色保持現在綠色。
8. 總件數顏色保持現在藍色。
9. 顏色不變。
10. 不要格內滾動。
11. 不要只顯示第一、二行。
12. 二、三、四、五排商品內容都要完整顯示。
13. 格子可依內容自動變高。
14. 同一排所有格子高度要一起對齊。
15. 不可只讓單一格變高導致整排歪掉。
16. A/B 區每一列要保持整齊。
17. 第一行和商品行都必須保持在同一格內。
18. 不准舊 renderer 改回舊格式。
19. 不可靠 DOM 文字反推件數，要從資料結構算。
20. 空格顯示「空格」。
21. 未指定客戶顯示「庫存」。
22. 不顯示 FOB / CNF / FOB代付。
23. 不顯示尺寸全文。

10.5 格子事件

點格子：

1. 開啟格位編輯。
2. 顯示目前格位。
3. 顯示此格目前商品。
4. 可搜尋已錄入商品。
5. 可批量加入商品。
6. 可刪除格內商品。
7. 可編輯備註。
8. 儲存後寫入 DB。
9. 儲存後前端立即更新。
10. 重新整理後仍在。

長按 / 右鍵：

開啟操作表：

1. 開啟/編輯格位
2. 插入格子
3. 刪除此格
4. 批量增加格子
5. 批量刪除格子
6. 標記問題格
7. 取消問題格
8. 返回該格
9. 關閉

10.6 操作表規則

1. 必須出現在畫面正中間。
2. 不是下拉選單。
3. 點功能後要關閉。
4. 點外面要關閉。
5. Esc 要關閉。
6. 手機和電腦都能用。
7. 拖拉時不可跳出長按選單。

10.7 批量加入商品

預設三列：

1. 後排
2. 中間
3. 前排

每列：

1. 下拉選未入倉商品
2. 加入按鈕

10.8 未入倉商品

來源：

1. 庫存
2. 訂單
3. 總單

規則：

1. A區只顯示 A區未入倉。
2. B區只顯示 B區未入倉。
3. 已入倉商品從下拉移除。
4. 刪除後回到下拉。
5. 下拉要顯示客戶、材質、尺寸、支數 x 件數。
6. 不要搜尋時一直重算未入倉。

10.9 新增 / 插入格子

1. 前端立刻顯示。
2. DB 同步寫入。
3. 後面格號往後移。
4. 不可覆蓋有商品格。
5. 不可重排有商品格。
6. 重新整理仍存在。
7. 失敗才還原或保留重送。

10.10 刪除格子

1. 只允許刪空格。
2. 有商品格不可刪。
3. 後面格號往前補。
4. DB 同步更新。
5. 前端立即更新。
6. 失敗要還原。

10.11 拖拉移動

1. 支援 pointer drag。
2. 不用 HTML5 drag。
3. 拖拉開始後取消長按。
4. 拖拉後前端立即更新。
5. DB 同步更新。
6. 失敗才還原。
7. 拖拉後件數重新計算。
8. 拖到已有商品格時，新移入商品要能標示 placement。

10.12 倉庫保存重大規則

倉庫所有長按功能：

1. 必須保存到 DB。
2. 重新整理不能消失。
3. 切頁不能中斷。
4. 關閉頁面後下次繼續保存。
5. 診斷必須能抓出未保存問題。

適用：

1. 單格新增
2. 批量增加格子
3. 刪除格子
4. 批量刪除格子
5. 編輯格子
6. 加入商品
7. 移除商品
8. 標記問題格
9. 取消問題格
10. 拖拉移動
11. 返回該格
12. 修改備註
