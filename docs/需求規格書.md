# 沅興木業｜乾淨重做系統完整需求規格書

> 目的：把目前 ZIP 內的 FIX138 母版功能，加上後續所有統整要求，整理成一份「乾淨重做」規格。新系統要直接用乾淨架構完成，不再靠舊版 `FIXxx hardlock` 疊加覆蓋，避免畫面亂跳、按鈕失效、出貨卡住、返回主頁很慢。

---

## 0. 重做原則

### 0.1 新系統要解決的根本問題
目前舊版問題不是單一按鈕壞掉，而是多個舊母版、舊函式、舊 CSS、舊 timer、舊 MutationObserver 互相覆蓋。新系統必須重新整理成單一乾淨母版：

1. **每個頁面只保留一套渲染函式**，不可同時有舊版和新版畫面互相重畫。
2. **每個功能只保留一套提交流程**，例如出貨只允許一個 `confirmShipment()` 主流程，不再有多個 `confirmSubmitXX` 互包。
3. **每個清單只保留一套資料來源**，例如庫存清單只從 `/api/inventory` 進入同一個資料轉換器。
4. **CSS 只保留一套設計系統**，不可再用 `fix135/fix136/fix137/fix138` 後蓋前。
5. **舊功能要被整合，不是直接刪掉**。能用的商業邏輯保留，舊版畫面與舊版重複覆蓋機制不保留。
6. **回首頁與開功能頁必須立即有反應**，不可因自動刷新、全頁重算、重複監聽器造成延遲。

### 0.2 新系統不可再使用的舊作法
新專案不要再照搬這些作法：

- 不再使用 `static/app.js` 內多段重複覆寫同一函式的方式。
- 不再用大量 `fixXXX_hardlock.js/css` 最後覆蓋舊版。
- 不再靠全頁 `MutationObserver` 不斷抓舊 DOM 後重畫。
- 不再用多層 `setTimeout` 延遲安裝功能。
- 不再讓 PWA 快取吃舊版 JS/CSS 後造成手機畫面不更新。
- 不再讓下拉選單、表格、小卡同時搶同一份商品資料。

乾淨重做建議分成這些固定模組：

## 1. 全站共同規格

### 1.1 品牌與風格

系統名稱固定為：**沅興木業**。

整體風格：簡約、清爽、有商業感、木業品牌感，但不要花俏。視覺方向：

- 背景：柔和米白、淡灰、淺木紋或低對比雲彩背景。
- 主色：深木色、深灰、米色。
- 按鈕：主要按鈕深色大按鈕，次要按鈕淺色小按鈕。
- 標籤：簡約淡灰色，不要黑色長條，不要過長，只要剛好包住文字。
- 卡片：柔和陰影、圓角一致、間距清楚。
- 字體：手機操作要大、清楚、避免誤觸。
- 錯誤訊息：用小卡片顯示，不用純文字紅字亂跳。
- 載入狀態：用 loading skeleton 或「載入中」卡片，不要空白。

### 1.2 手機優先

必須先確保 iPhone / Android 手機直向可用，再支援橫向與平板。

- 按鈕高度至少 44px。
- 列表行距要夠大，手指容易點。
- 輸入框、下拉、搜尋都要適合手機。
- 表格在手機上可橫滑，但不能整頁卡住。
- 倉庫圖可左右滑、上下滑，不可點一下卡住。
- 拍照與上傳要使用手機原生相機 / 相簿體驗。

### 1.3 導航與速度

- 首頁 → 功能頁：立即切換，不可等 2～3 秒才有畫面。
- 功能頁 → 返回首頁：立即返回，不可卡住。
- 頁面打開時只載必要資料，不可全站一次重算。
- 資料更新採「局部刷新」：
  - 倉庫更新只刷新倉庫區。
  - 今日異動更新只刷新今日異動清單與 badge。
  - 客戶資料更新只刷新客戶區。
  - 商品更新只刷新該模組商品清單。

### 1.4 即時同步與通知

多使用者操作時，其他人要看到同步更新，但不可因此卡頓。

通知分三層：

1. **短 toast / banner**：新增、編輯、出貨、儲存成功時顯示 2～3 秒。
2. **今日異動詳細列表**：顯示誰、何時、做了什麼、影響哪個客戶 / 商品 / 格位。
3. **今日異動 badge**：首頁「今日異動」顯示紅點或數字；打開今日異動後歸零，新異動再出現。

## 4. 商品資料共通規格

### 4.1 商品文字格式

商品固定使用這種格式：

```text
長x寬x高=支數x件數+支數+支數
```

範例：

```text
61x12x10=720x2
132x23x05=249x3
132x49x04=304x2+114+112+33
100x30x63=504x5+588+587+502+420+382+378+280+254+237+174
```

### 4.2 符號標準化

輸入時要自動轉成統一格式：

- `×`、`Ｘ`、`X`、`✕`、`＊`、`*` → `x`
- `＝` → `=`
- `，`、`,`、`；`、`;`、`＋` → `+`
- 多餘空白移除。
- 高度單碼要補 0，例如 `5` → `05`。
- `0.83` / `.83` 要存成 `083`。
- `063`、`083` 這種前導 0 不可刪掉。

### 4.3 底線承接上一筆寬高

如果遇到：

```text
120x33x33=70x4
159x33x165=131x5
179x___=131x4
```

要自動承接上一筆寬高，輸出為：

```text
120x33x33=70x4
159x33x165=131x5
179x33x165=131x4
```

### 4.4 件數判定規則

件數不能再用手動輸入為主，必須由商品右側自動判斷。

規則：

1. `132x23x05=249x3` → 3 件。
2. `132x23x05=249` → 1 件。
3. `60+54+50` → 3 件。
4. `220x4+223x2+44+35+221` → 9 件。
5. `100x30x63` 沒有等號 → 1 件，不可誤判為 63 件。
6. `100x30x63=504x5+588+587+502+420+382+378+280+254+237+174` → 15 件。

### 4.5 材質規則

商品資料要把材質和尺寸分開存：

- `material`：材質，例如 白鐵、尤佳利、杉木等。
- `product_text`：尺寸與支數件數。
- 如果材質欄位看起來像商品尺寸，例如含有 `x` 或 `=`，不可當材質。
- `尤加利 / 尤佳利` 都要支援，顯示建議統一成常用名稱。

### 4.6 排序規則

商品列表排序：

1. 材質。
2. 月份，如果商品前面有 `1月`～`12月`。
3. 高，由小到大。
4. 寬，由小到大。
5. 長，由小到大。
6. 同商品再依件數、支數排序。

總單統整如有月份，應優先照月份 → 高 → 寬 → 長，讓客戶對帳更直覺。

## 9. 出貨頁規格

### 9.1 頁面用途

出貨頁從客戶商品中選取要出貨的商品，先顯示出貨預覽，再確認扣除。

### 9.2 客戶選擇

- 客戶名稱輸入第一個字就出現既有客戶建議。
- 支援北 / 中 / 南客戶卡片點選。
- 出貨頁要有「客戶快速選擇」區，不可載不到客戶名。
- 點客戶後立即載入該客戶商品。

### 9.3 客戶商品清單

出貨頁不要只依賴會亂跳的下拉選單，要顯示完整直列清單：

每筆顯示：

- 來源：總單 / 訂單 / 庫存
- 材質
- 尺寸
- 支數 x 件數
- 件數
- 加入按鈕

按鈕：

- 加入選取商品
- 整個加入下方商品資料
- 重新載入
- 一鍵清空已選商品

### 9.4 已選商品

已選商品要能：

- 修改本次出貨數量 / 支數。
- 刪除單筆。
- 一鍵清空。
- 同商品自動合併。
- 保留來源資訊，避免扣錯來源。

### 9.5 出貨預覽

按確認送出後，不能直接扣，要先顯示出貨預覽。

預覽必須顯示：

1. 本次出貨件數。
2. 每筆商品明細。
3. 商品倉庫圖位置。
4. 材積算式。
5. 材積合計。
6. 重量輸入框。
7. 總重 = 材積 × 重量。
8. 扣除來源：總單 / 訂單 / 庫存。
9. 扣除前數量 → 扣除後數量。
10. 不足提醒。
11. 借貨提示。
12. 取消 / 確認扣除。

### 9.6 出貨扣除邏輯

出貨來源邏輯：

1. 如果商品是從總單列按「直接出貨」，扣總單。
2. 如果商品是從訂單列按「直接出貨」，扣訂單。
3. 如果商品是從庫存列按「直接出貨」，扣庫存。
4. 如果是出貨頁自動選商品，優先扣總單；總單不足再看訂單；再看庫存。
5. 如果使用者選定來源，必須照指定來源扣，不可自動改。
6. 如果都不足，不可扣除，要顯示不足原因。

### 9.7 借貨邏輯

如果使用者選到的商品不屬於目前出貨客戶，而其他客戶有該商品，要提示：

```text
該客戶沒有這筆商品，是否向 xxx 借 xxx尺寸 = xxx支 x 幾件？
```

使用者確認後：

- 出貨記錄仍記在本次出貨客戶。
- 實際扣除來源記錄為借貨客戶。
- 預覽與完成摘要要顯示「向 xxx 借貨」。

### 9.8 出貨完成

確認扣除後要顯示：

- 出貨完成。
- 每筆商品扣除來源。
- 實際扣總單 / 訂單 / 庫存多少。
- 扣除前 → 扣除後。
- 如有借貨，顯示借貨客戶。
- 同步刷新總單 / 訂單 / 庫存 / 今日異動 / 倉庫未入倉數。

## 10. 材積、長度、重量算式

### 10.1 支數總和

等號右側代表支數或長度段：

```text
80x30x125=111+132x3
```

支數總和為：

```text
111 + 132×3
```

### 10.2 尺寸轉換

材積計算使用：

```text
支數總和 × 長度係數 × 寬度係數 × 高度係數
```

長度係數：

- 長度 > 210，代表超過 210，要用 `長度 / 1000`，例如 `363 → 0.363`、`544 → 0.544`、`212 → 0.212`。
- 長度 ≤ 210，用 `長度 / 100`，例如 `80 → 0.8`、`140 → 1.4`。

寬度係數：

- `30 → 3`
- `25 → 2.5`
- 公式為 `寬 / 10`。

高度係數：

- `125 → 1.25`
- `12 → 1.2`
- `05 → 0.5`
- `083 → 0.83`
- 若高度數字 ≥ 100，使用 `高 / 100`；否則使用 `高 / 10`。

### 10.3 材積範例

商品：

```text
80x30x125=111+132x3
140x30x12=294x2+10
363x25x05=100
```

材積算式：

```text
(111 + 132×3) × 0.8 × 3 × 1.25
+ (294×2 + 10) × 1.4 × 3 × 1.2
+ 100 × 0.363 × 2.5 × 0.5
= 材積
```

出貨預覽要逐筆顯示算式，也要顯示合計。

### 10.4 重量

出貨預覽旁邊要有重量輸入框：

```text
總重 = 材積合計 × 輸入重量
```

輸入重量後即時更新總重。

## 12. 倉庫圖規格

### 12.1 基本結構

倉庫圖分 A 區 / B 區。

每區：

- 6 欄。
- 每欄為垂直長條。
- 格位數可動態增加 / 刪除。
- 原始概念為每欄 1～20 格，但不能硬限制死 20 格。
- 支援全部顯示 / A 區 / B 區。
- 記住上次停留區域。

### 12.2 倉庫格顯示格式

格子裡只顯示兩行，不顯示 FOB / CNF / 尺寸商品資訊。

固定格式：

```text
1  力凡/永和/保固
4+2+1                    7件
```

規則：

- 第一行左側只顯示格號數字，例如 `1`，不要顯示「第01格」。
- 第一行右側顯示客戶名稱，可多個客戶用 `/` 連接。
- 如果未指定客戶，顯示 `庫存`。
- 不顯示 `FOB`、`CNF`、`FOB代付`。
- 不顯示尺寸，例如 `132x32x12`。
- 第二行左側顯示每筆件數相加，例如 `4+2+1`。
- 第二行最右側顯示總件數，例如 `7件`。
- 客戶名稱用紅字。
- 件數與加總用藍字。

### 12.3 倉庫格操作

點格子：開啟 / 編輯格位。

長按或右鍵格子：開啟格位操作表：

- 開啟 / 編輯格位
- 在此格後插入格子
- 刪除此格

刪除格位規則：

- 格子內有商品時不可刪除。
- 必須先移除商品才可刪除。
- 刪除後同欄後方格號自動往前補。

### 12.4 格位編輯視窗

格位視窗要顯示：

- 目前格位：A 區第幾欄第幾格。
- 目前此格商品清單。
- 每筆商品顯示客戶 / 商品文字 / 件數。
- 可移除單筆。
- 可加入未錄入倉庫圖商品。
- 可搜尋商品。
- 可輸入加入件數。
- 儲存格位。

### 12.5 未入倉 / 未錄入倉庫圖

倉庫頁上方顯示：

```text
未入倉：X件
```

規則：

- 名稱用「未錄入倉庫圖」或「未入倉」都可，但列表標題要用「未錄入倉庫圖」。
- 只顯示總件數，不要在件數 / 筆數之間切換。
- 不要自動一直重算，避免卡頓。
- 旁邊保留刷新按鈕，使用者按了才重新抓未錄入數量。
- A 區只取 A 區商品，B 區只取 B 區商品。

### 12.6 高亮與搜尋

倉庫圖要有：

- 搜尋商品 / 客戶 / 格位。
- 搜到後自動捲到格位並閃爍。
- 同客戶高亮。
- 未入倉高亮。
- 清除高亮。
- 點訂單商品可跳到倉庫圖並閃爍該商品位置。

### 12.7 拖拉移動

- 支援滑鼠拖拉。
- 支援手機 pointer 拖拉。
- 拖拉開始移動後，不可又跳出長按操作表。
- 拖到已有商品格時，新增商品放在「前排」。
- 後端記錄 `placement_label = 前排`。
- 移動後只刷新倉庫區，不全頁刷新。
- 支援「還原上一步」。

## 13. 客戶資料頁規格

### 13.1 客戶資料欄位

每個客戶資料包含：

- 客戶名稱
- 電話
- 地址
- 特殊要求
- 常用材質
- 常用尺寸
- 區域：北區 / 中區 / 南區
- UID
- 是否封存

### 13.2 客戶列表

客戶以北 / 中 / 南分類。

客戶卡片同樣使用：

```text
客戶名稱        CNF/FOB/FOB代        件數 / 筆數
```

### 13.3 客戶操作

- 點選：載入客戶資料。
- 長按 / 右鍵：操作表。
- 編輯後立即刷新，不要返回再打開才看到。
- 移區後立即顯示到新區。
- 封存後不顯示於一般列表。
- 可從封存客戶列表還原。
- 刪除需確認。

### 13.4 客戶同步

系統不要有獨立「新增客戶」按鈕作為唯一入口。客戶應該可由訂單 / 總單 / 出貨 / 入庫自動同步進客戶資料，且仍可人工編輯。

客戶名稱改名時：

- 要同步 inventory / orders / master_orders / shipping_records。
- 以 UID 做主關聯，名稱只當顯示。
- 需保留別名與模糊比對。

## 20. 驗收清單

### 20.1 速度

- 首頁點庫存、訂單、總單、出貨、倉庫圖，1 秒內要看到頁面骨架。
- 返回首頁不可卡住。
- 點客戶後商品立即出現。
- 出貨頁客戶商品清單不可亂跳。

### 20.2 功能

- 庫存可新增、編輯、刪除、批量材質、批量刪除、加到訂單、加到總單。
- 訂單只顯示有訂單客戶。
- 總單只顯示有總單客戶，商品列不重複顯示客戶名。
- 出貨可預覽、可算材積、可輸入重量、可扣總單 / 訂單 / 庫存。
- 出貨不足時不扣除。
- 借貨流程要有提示與紀錄。
- 倉庫圖 A/B 區可切換，格子格式正確。
- 倉庫格可長按插入 / 刪除。
- 倉庫可拖拉到前排。
- 今日異動 badge 打開後歸零。
- 今日異動可滑動刪除並立即消失。
- 設定頁管理員功能不可 500。
- 還原上一步可用。

### 20.3 商品規則

以下測試必須通過：

```text
132x23x05=249x3                         => 3件
132x23x05=249                            => 1件
60+54+50                                 => 3件
220x4+223x2+44+35+221                    => 9件
100x30x63                                => 1件
100x30x63=504x5+588+587+502+420+382+378+280+254+237+174 => 15件
179x___=131x4                            => 自動承接上一筆寬高
0.83                                     => 083
063                                      => 保留 063
```

### 20.4 舊版覆蓋防止

乾淨版驗收時，專案內不應再有：

```text
fix135_master_final_hardlock.js
fix136_label_text_repair.js
fix137_undo_layout_warehouse_hardlock.js
fix138_final_master_hardlock.js
legacy_isolation_hardlock.js
多個 confirmSubmitXX
多個 renderWarehouseXX
多個 product action hardlock
```

這些只能當參考，不能當主線載入。

新系統要做成：

**一套乾淨、單一母版、手機優先、不卡頓、不被舊版覆蓋的沅興木業庫存 / 訂單 / 總單 / 出貨 / 倉庫圖系統；所有商品解析、件數、材積、借貨、扣除、倉庫格位、今日異動、客戶資料與管理員權限都要用同一套核心邏輯統一處理。**
